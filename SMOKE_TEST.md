# SMOKE_TEST — מעבר ידני מלא

Manual walkthrough for release checks. Run `npm run dev` (or `npm run dev:web` in a browser).

1. פתח את האפליקציה — מסך הכניסה "מה-בלוז" מוצג, עברית RTL, ללא אנגלית.
2. בחר "סגן נועם לוי — מפקד הכשרה" ולחץ "כניסה למערכת".
3. בדשבורד: מוצגת "הכשרת מודיעין מחזור 24", מצב טיוטה, רכיבים קשיחים קרובים, פעולות מהירות.
4. עבור ל"ייבוא מאקסל" — טען קובץ לדוגמה, לחץ "נתח קובץ", ודא שמסך "סקירת ייבוא" מציג רכיבים שזוהו עם רמות ביטחון, ושמור.
5. עבור ל"לו״זים משותפים" — ודא שקיים "הרצאת בטיחות" מקושר לשתי הכשרות, ושקיימת בקשת שינוי ממתינה. אשר או דחה אותה (בדחייה: מוצג "שינוי תקוע" + "הצג פרטי קשר").
6. עבור ל"ימי שיא" — הוסף יום שיא מסוג "מטווחים". אם יש התנגשות עם הרצאת חוץ/לו״ז משותף — מוצגת אזהרה מתאימה לפני ההמשך.
7. עבור ל"מרצי חוץ" — הוסף הרצאת חוץ למחר (בחר מרצה קיים או צור חדש). ודא סטטוס "טרם נשלחה תזכורת".
8. עבור ל"בניית לו״ז" — לחץ "הפק לו״ז", עיין בתוצאה (קונפליקטים, רכיבים שלא שובצו, הצעות), החל על הטיוטה.
9. בדוק גרירת רכיב גמיש בתצוגה השבועית — רכיב נעול לא ניתן לגרירה.
10. לחץ "פרסם לו״ז" — ודא שמוצג דוח השפעות שינוי; אם יש קונפליקטים חוסמים נדרש אישור חריג; אשר ופרסם.
11. לחץ "עבור לתצוגת חייל" (או התנתק ובחר "רב״ט יואב כהן") — ודא שמוצג רק הלו״ז שפורסם, ללא כפתורי עריכה, ושרכיבים מוסתרים אינם מוצגים.
12. התנתק ובחר "רס״ן דניאל מזרחי — מפקד בכיר" — ודא דשבורד עם שתי הכשרות, מונים, וכניסה לכל הכשרה.
13. עבור ל"מוקד הודעות" — לחץ "הפעל בדיקת תזכורות". ודא שנרשמה "תזכורת נשלחה" להרצאה של מחר עם נושא בעברית.
14. פתח את קישור האישור של המרצה (העתק קישור אישור מההרצאה: ‎#/lecturer-confirmation/‎<token>‎) — ודא "המרצה אישר הגעה להרצאה." ושנוצרה התראה למפקד.
15. חזור ל"מוקד הודעות" — סטטוס אישור הגעה עודכן.
16. עבור ל"טיוטות וגרסאות" — מוצגות טיוטה, לו״ז נוכחי וגרסאות קודמות; שחזור חסום כשיש שינוי בלו״ז משותף.
17. עבור ל"מרכז קונפליקטים" — קונפליקטים מוצגים עם הצעות; הפעל פתרון מומלץ ובדוק שהקונפליקט נסגר.
18. התנתק ובחר "מנהל מערכת ARTECH" — ודא "אזור מנהל": משתמשים, הגדרות תכונות (עתידיות כבויות), עריכת שאלות נפוצות, יומני מערכת.
19. בדוק "שאלות נפוצות" ו"יצירת קשר" (טלפון תמיכה מ-VITE_SUPPORT_PHONE בלבד).
20. ודא שאין טקסט אנגלי במסכים (למעט מונחים כמו PDF/Excel) ואין אמוג׳ים.
